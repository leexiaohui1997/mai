# 单元测试助手 (Uni Test Helper)

## 执行流程

### 步骤 0：确定测试目标范围

- **用户指定文件** → 直接进入步骤 2，针对指定文件编写测试
- **未指定文件** → 进入步骤 1，自动检测变更文件

### 步骤 1：自动检测变更文件

**执行命令：**

```bash
git status --porcelain
```

**变更状态解析：**

- **无输出** → 无文件变更，流程终止
- **有输出** → 分析变更状态：
  - **首字符非空格** → 暂存区变更（Modified/Added/Deleted/Renamed）
  - **首字符为空格** → 工作区变更（未暂存）
  - **??** → 未跟踪的新文件

**处理逻辑：**

- 存在暂存区变更 → 测试范围 = 所有暂存文件
- 仅工作区变更 → 测试范围 = 所有工作区变更文件

### 步骤 2：单元测试编写流程

对每个目标文件执行以下子流程：

#### 2.1 创建测试文件

在源文件同级目录创建对应的测试文件：

```
source.js → source.test.js
module.ts → module.test.ts
```

若测试文件已存在，跳过创建步骤

#### 2.2 确定测试范围

- **用户指定文件** → 全量测试
- **自动检测文件** → 增量测试（仅测试变更部分）

**变更分析命令：**

```bash
# 工作区变更详情
git diff <filename>

# 暂存区变更详情
git diff --cached <filename>
```

#### 2.3 编写测试代码

- **测试框架**: Vitest 4.1.2
- **语言**: 使用中文编写测试描述
- **范围**: 根据 2.2 确定的测试范围编写针对性测试

#### 2.4 测试验证

- 识别测试文件所属的子项目
- 执行单文件测试：

```bash
# 在根目录执行
pnpm -F <subproject> run test -- <test-file-path>
```

- **测试失败** → 返回步骤 2.3 调整测试代码
- **测试通过** → 处理下一个文件
- **所有文件完成** → 进入步骤 3

### 步骤 3：全面验证与覆盖率报告

在项目根目录执行完整测试套件：

```bash
pnpm run test:coverage
```

生成覆盖率报告，验证整体测试质量

## 问题与经验

### 1. 导入缺失问题

- 问题: 测试中使用了 vi 和 beforeEach 但未导入
- 解决方案: 确保导入所有需要的 Vitest 工具函数
- 经验: 在编写测试前检查现有测试文件的导入模式
